The Challenge

Kircha a centuries-old Ethiopian communal practice of collectively purchasing, slaughtering, and equitably sharing a bull or ox during holidays functions as a traditional food security mechanism. By pooling resources, low- and middle-income households access fresh, affordable animal protein in a country.Despite its cultural, social, and economic importance, the traditional practice of Kircha in Ethiopia is under threat. Rising cattle prices have made it increasingly unaffordable for many groups to collectively purchase animals, limiting access to meat for low- and middle-income households. Urbanization and frequent relocation of community members have fragmented the close-knit social networks essential for organizing Kircha, weakening community cohesion.


How might we improve access to affordable, nutritious animal protein for low- and middle-income Ethiopian communities, while addressing the economic hardships, food price inflation, and ongoing crises that undermine food security and nutritional health?

Our Solution

To modernize and digitize the Ethiopian Kircha tradition by enabling community members to easily create or join Kircha groups with transparency, cultural respect, affordability, and flexible service options, while supporting local farmers and maintaining social cohesion.


The Kircha PWA features two distinct user flows:

  • Create Kircha Group: Organizers select verified farmer-listed oxen or bulls, set culturally sensitive slaughter dates and times, and choose slaughtering method (self or company-managed).
  • Join Kircha Group: Users join company-stocked Kircha groups with slaughter dates and times displayed prominently to respect fasting practices.

Key Stakeholders

  • Kircha Organizers: Users who create Kircha groups, select farmers' livestock, set slaughter schedules, and manage member onboarding.
  • Kircha Members: Participants who join existing groups, provide delivery details, and make payments.
  • Livestock Farmers: Verified farmers who list oxen/bulls, manage availability, and coordinate livestock delivery.
  • Kircha Company: Provides livestock for joiners, manages slaughtering and delivery, and handles payment processing.
  • Community Members: Christian users observing cultural and fasting practices related to Kircha.
  • Developers & Admins: Manage platform infrastructure, security, and user support.

System Overview

Frontend (PWA): Installable, offline-capable web app with service worker and manifest. Users create/join Kircha groups, browse livestock, and pay via mobile. Backend (Django): Handles user auth, group logic, database (PostgreSQL), and REST APIs. Manages real-time availability and payment tracking. Payments (Daraja API): STK Push sends M-Pesa prompt to user’s phone. Callbacks update payment status instantly.

Installation Guide

Step-by-step setup for developers. All commands are run in a terminal.

1. Clone repository
  git clone https://github.com/Tihitnaabraham/meet-for-meat.git
  cd meet-for-meat

2. Create virtual environment
  python -m venv venv
  source venv/bin/activate   # Windows: venv\Scripts\activate

3. Install dependencies
  pip install -r requirements.txt

4. Apply migrations
  python manage.py makemigrations
  python manage.py migrate

5. Create superuser
  python manage.py createsuperuser

6. Run server
  python manage.py runserver
Tip: Use python -m pip install --upgrade pip before step 3 if you encounter version conflicts.

Configuration Guide

Environment variables and external services required for full functionality.

Database

PostgreSQL is recommended for production. For local development you may keep the default SQLite.

DATABASE_URL=postgres://user:pass@localhost:5432/meetformeat

M-Pesa (Daraja)

Create a .env file in the project root and add:

CONSUMER_KEY=your_consumer_key
CONSUMER_SECRET=your_consumer_secret
SHORTCODE=174379
PASSKEY=your_passkey

PWA Manifest

Edit static/manifest.json and replace start_url and scope with your production domain.

Important: Never commit .env to version control. Add it to .gitignore.

Glossary

Key terms used throughout the documentation.

Kircha
Traditional Ethiopian communal meat-sharing practice where a group collectively buys, slaughters, and divides an animal.
STK Push
Daraja M-Pesa prompt that appears on a user’s phone to confirm a payment.
PWA
Progressive Web App – a web application that can be installed on a device, works offline, and feels like a native app.
Daraja
Safaricom’s official API suite for M-Pesa integrations (C2B, B2C, STK Push, etc.).
Half Kircha
Group size ≤ 20 members; each member receives half a share of the meat.
Full Kircha
Group size ≤ 10 members; each member receives a full share of the meat.
JWT
JSON Web Token – stateless authentication mechanism used for API access.

Key Findings

  • Economic hardship from rising cattle prices impacts group affordability.
  • Social fragmentation due to urbanization weakens trust and group formation.
  • Youth disinterest in traditional, time-consuming Kircha processes.
  • High cultural importance noted by elders for identity and food security.
  • Need for transparency and reliable payment and livestock confirmation.
